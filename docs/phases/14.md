## Problem

### The Latency Problem

When typing over SSH:
1. Key pressed locally
2. Key sent to server (latency: 100ms)
3. Server processes (small delay)
4. Server echoes back (latency: 100ms)
5. Total round-trip: **200ms+**

This makes typing feel sluggish and unresponsive.

### Solution: Predictive Echo

Show the predicted character immediately:
1. Key pressed locally
2. **Prediction shown immediately**
3. Key sent to server
4. Server echoes back
5. **Compare prediction to echo**
6. **Correct if wrong**

Result: **Perceived latency: ~0ms**

## Architecture

### Prediction Engine

```rust
struct PredictionEngine {
    enabled: bool,
    predictions: Vec<Prediction>,
    last_sent: Option<char>,
}

struct Prediction {
    position: Position,
    predicted_char: char,
    sent_time: Instant,
    state: PredictionState,
}

enum PredictionState {
    Pending,        // Waiting for echo
    Confirmed,      // Echo matched
    Corrected,      // Echo different, corrected
    Undone,         // Rolled back before echo
}

impl PredictionEngine {
    fn on_key_press(&mut self, c: char) -> Option<Prediction> {
        if !self.enabled {
            return None;
        }
        
        // Common cases where we can safely predict:
        // 1. Regular character in normal mode
        // 2. Shell prompt typing
        
        if self.is_predictable(c) {
            let prediction = Prediction {
                position: self.cursor_position(),
                predicted_char: c,
                sent_time: Instant::now(),
                state: PredictionState::Pending,
            };
            self.predictions.push(prediction);
            Some(prediction)
        } else {
            None
        }
    }
    
    fn on_receive(&mut self, data: &[u8]) {
        for prediction in &mut self.predictions {
            if prediction.state == PredictionState::Pending {
                // Check if echo matches
                if self.matches_prediction(prediction, data) {
                    prediction.state = PredictionState::Confirmed;
                } else {
                    prediction.state = PredictionState::Corrected;
                    self.undo_prediction(prediction);
                }
            }
        }
    }
}
```

impl PredictionEngine {
    fn undo_prediction(&self, prediction: &Prediction) {
        // Remove the predicted character
        self.grid.move_to(prediction.position);
        self.grid.delete_char();
        
        // Process the actual echo
        // (passed separately)
    }
    
    fn matches_prediction(&self, prediction: &Prediction, data: &[u8]) -> bool {
        // Check if the received data matches what we predicted
        // This is tricky - we need to parse the output and find
        // where our predicted character should appear
        
        // Simple case: direct character match
        if data.len() == 1 && data[0] as char == prediction.predicted_char {
            return true;
        }
        
        // Complex case: echo might contain escape sequences
        // We need to parse and find the actual character
        let parsed = self.parser.parse(data);
        if let Some(char_output) = self.extract_character(&parsed) {
            return char_output == prediction.predicted_char;
        }
        
        false
    }
}

## Predictable Cases

### When to Predict

1. **Regular typing at shell prompt**
   - Most characters are echoed directly
   - Safe to predict alphanumeric, symbols

2. **Line editing in shell**
   - Characters appear at cursor
   - Backspace deletes previous character

### When NOT to Predict

1. **Password input**
   - Echo is suppressed
   - Prediction would leak password

2. **Control sequences**
   - Arrow keys move cursor
   - Tab triggers completion
   - Ctrl+C interrupts

3. **Programs using raw mode**
   - Vim, less, tmux
   - Echo is disabled
   - Input handled differently

4. **Escape sequences in output**
   - Colors, cursor movement
   - Can't predict exact character position

### Detection

```rust
impl PredictionEngine {
    fn is_predictable(&self, c: char) -> bool {
        // Check if we're in a predictable context
        if !self.at_shell_prompt() {
            return false;
        }
        
        // Check if character is safe to predict
        match c {
            // Regular characters
            'a'..='z' | 'A'..='Z' | '0'..='9' => true,
            // Common symbols
            ' ' | '!'..='~' => true,
            // NOT predictable
            '\n' | '\r' | '\t' => false,
            '\x01'..='\x1f' => false, // Control characters
            _ => false,
        }
    }
    
    fn at_shell_prompt(&self) -> bool {
        // Use OSC 133 marks to detect prompt
        // Or detect common prompt patterns
        // Or track when alternate buffer is active
        
        !self.in_alternate_buffer && 
        self.last_mark == PromptMark::PromptStart
    }
}
```

## SSH Optimizations

### Latency Detection

```rust
struct ConnectionMetrics {
    rtt_estimate: Duration,
    last_ping: Instant,
    last_pong: Instant,
}

impl ConnectionMetrics {
    fn estimate_rtt(&mut self) -> Duration {
        // Calculate from ping/pong times
        // Or from SSH protocol
        self.rtt_estimate
    }
    
    fn should_predict(&self) -> bool {
        // Only predict if latency is noticeable
        self.rtt_estimate > Duration::from_millis(50)
    }
}
```

### Adaptive Prediction

```rust
impl PredictionEngine {
    fn adapt(&mut self, metrics: &ConnectionMetrics) {
        // Enable/disable based on measured latency
        if metrics.rtt_estimate > Duration::from_millis(50) {
            self.enabled = true;
        } else {
            self.enabled = false;
        }
    }
}
```
