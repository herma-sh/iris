# Phase 2: wgpu Renderer

**Duration**: 2-3 weeks  
**Priority**: P0 Critical  
**Dependencies**: Phase 1  
**Goal**: GPU-accelerated rendering with 60fps scrolling

---

## Overview

This phase implements the GPU renderer using wgpu. The renderer takes the terminal grid and displays it on screen using the GPU. Key features: texture atlas for glyphs, efficient batching, damage-only rendering, and 60fps scrolling even with millions of lines of scrollback.

## Crate Structure

`iris-render-wgpu` is split into low-level GPU primitives plus two stateful renderer layers (`TextRenderer` and `TerminalRenderer`) that power retained-frame updates.

```text
iris-render-wgpu/
|- Cargo.toml
`- src/
   |- lib.rs
   |- error.rs
   |- renderer.rs
   |- surface.rs
   |- texture.rs
   |- pipeline.rs
   |- atlas.rs
   |- glyph.rs
   |- font.rs
   |- cell.rs
   |- cursor.rs
   |- theme.rs
   |- text_renderer.rs
   |- terminal_renderer.rs
   `- shaders/
      |- text.wgsl
      |- cursor.wgsl
      |- present.wgsl
      `- fullscreen.wgsl
```

Module responsibilities:

- `renderer.rs`: owns `wgpu` instance/adapter/device/queue, creates surfaces/resources/pipelines, and provides shared upload/draw helpers.
- `surface.rs` + `texture.rs`: validated presentation and off-screen render targets.
- `pipeline.rs`: fullscreen bootstrap, text, cursor, and retained-frame present pipelines.
- `atlas.rs` + `glyph.rs`: atlas allocation/upload plus cache residency and placement-aware cache invariants.
- `font.rs`: system font discovery, fallback, rasterization, and shared-baseline glyph placement offsets.
- `cell.rs` + `cursor.rs`: instance encoding and GPU buffer layouts.
- `text_renderer.rs`: stateful atlas/cache/buffer orchestration for damage-driven text+cursor preparation.
- `terminal_renderer.rs`: terminal-facing retained frame lifecycle with full prepare, incremental updates, scroll shifting, and presentation.
- `theme.rs`: renderer theme model, color resolution, and TOML loading.

## API Contracts

### Public Crate Interface (`lib.rs`)

The crate exports both foundational GPU primitives and higher-level renderer state machines:

```rust
pub use renderer::{Renderer, RendererConfig};
pub use text_renderer::{TextRenderer, TextRendererConfig};
pub use terminal_renderer::{TerminalRenderer, TerminalRendererConfig};
pub use font::{FontRasterizer, FontRasterizerConfig};
pub use glyph::{CachedGlyph, GlyphCache, GlyphKey, GlyphPlacement, RasterizedGlyph};
pub use theme::{Theme, ThemeColor, ThemeLoadError};
```

### `Renderer` Contract (Resource Factory + Submission)

`Renderer` is the single owner of `wgpu` bootstrap state and is responsible for:

- creating/validating surfaces and off-screen textures,
- creating pipelines/atlas/buffers/caches,
- writing uniforms/instances and submitting render passes.

It is intentionally stateless about terminal content; retained state lives in higher-level renderers.

### `TextRenderer` Contract (Stateful Damage Prepare)

`TextRenderer` owns glyph atlas/cache + text/cursor buffers and prepares draws from `iris-core::Grid` damage regions.

Core lifecycle:

1. `prepare_grid*` populates missing glyphs, including cached rasterization offsets (`GlyphPlacement`), and uploads normalized-damage instances.
2. `prepare_cursor` uploads cursor overlay geometry.
3. `render_to_texture_surface` or `render_to_surface` issues text + cursor passes.

Guarantees:

- wide-cell continuation damage is normalized to include lead cells,
- blank but styled cells can still clear stale retained content,
- glyph cache reuse requires matching key, dimensions, and placement.

### `TerminalRenderer` Contract (Retained Frame + Incremental Updates)

`TerminalRenderer` integrates `TextRenderer` + `FontRasterizer` and owns a cached frame surface used for presentation.

Core lifecycle:

1. `prepare_terminal` performs a full visible-frame build into the retained frame.
2. `update_terminal` applies incremental terminal changes from damage + cursor invalidation.
3. scroll deltas shift the retained frame (full-grid or partial scroll-region) before redraw.
4. `render_to_*` presents the retained frame using `PresentPipeline`, including scroll-offset-aware sampling.

Failure and invalidation guarantees:

- if incremental prepare fails, retained frame state is invalidated,
- drained terminal damage/scroll deltas are restored on error,
- theme/uniform/font-size changes invalidate retained caches so the next update rebuilds from a full prepare.

### Placement and Cache Invariants

Placement is part of the renderer contract in this phase:

- rasterization produces per-glyph `GlyphPlacement` offsets,
- cache insertion is placement-aware (mismatched reinsertion returns an error),
- instance encoding carries `glyph_offset`/`glyph_extent`,
- `text.wgsl` samples atlas coverage only inside the glyph rectangle, preserving background outside glyph bounds.

## Notes

### Design Decisions

1. **R8Unorm texture format**: Single channel for glyph mask, enough for text rendering
2. **Row-based atlas allocation**: Simple, fast, good cache locality
3. **Instance-based rendering**: One instance per cell, reduces vertex data
4. **Damage-only rendering**: Only update changed cells

### Performance Considerations

1. **Glyph caching**: Cache rendered glyphs to avoid re-rasterization
2. **Atlas packing**: Tight packing of glyphs in texture atlas
3. **Batch rendering**: Single draw call for all cells
4. **Uniform buffer**: Update once per frame, not per cell

### Future Optimizations

1. **Multiple atlases**: For unicode ranges, reduce texture size
2. **Distance fields**: For better scaling
3. **Subpixel rendering**: For sharper text
4. **Compute shaders**: For font rendering
