## Color Depth

### Standard (24-bit)

```
R: 8 bits (0-255)
G: 8 bits (0-255)
B: 8 bits (0-255)
Total: 16.7 million colors
```

### Deep Color (30-36 bit)

```
10-bit:  R: 10 bits (0-1023)
         G: 10 bits (0-1023)
         B: 10 bits (0-1023)
         Total: 1.07 billion colors

12-bit:  R: 12 bits (0-4095)
         G: 12 bits (0-4095)
         B: 12 bits (0-4095)
         Total: 68.7 billion colors
```

### Why Deep Color?

1. **HDR displays**: Modern displays support >8-bit per channel
2. **Gradient banding**: 8-bit shows visible banding in gradients
3. **Color accuracy**: More precision for color-critical work
4. **Future-proof**: HDR is becoming standard

## Implementation

### Extended Color Struct

```rust
#[derive(Clone, Copy, Debug, PartialEq)]
pub struct Color {
    pub r: u16,
    pub g: u16,
    pub b: u16,
    pub a: u16,
    pub space: ColorSpace,
}

#[derive(Clone, Copy, Debug, PartialEq)]
pub enum ColorSpace {
    Srgb,
    DisplayP3,
    Rec2020,
}

impl Color {
    pub fn new_rgba(r: u16, g: u16, b: u16, a: u16) -> Self {
        Self { r, g, b, a, space: ColorSpace::Srgb }
    }
    
    pub fn with_space(mut self, space: ColorSpace) -> Self {
        self.space = space;
        self
    }
    
    pub fn to_srgb(&self) -> Self {
        match self.space {
            ColorSpace::Srgb => *self,
            ColorSpace::DisplayP3 => self.convert_p3_to_srgb(),
            ColorSpace::Rec2020 => self.convert_rec2020_to_srgb(),
        }
    }
    
    pub fn to_u8(&self) -> [u8; 4] {
        // Convert to 8-bit for fallback
        [
            (self.r >> 8) as u8,
            (self.g >> 8) as u8,
            (self.b >> 8) as u8,
            (self.a >> 8) as u8,
        ]
    }
}
```

### Color Space Conversions

```rust
impl Color {
    fn convert_p3_to_srgb(&self) -> Self {
        // Display-P3 -> XYZ -> sRGB
        let xyz = self.p3_to_xyz();
        let srgb = xyz_to_srgb(&xyz);
        Color { space: ColorSpace::Srgb, ..srgb }
    }
    
    fn convert_rec2020_to_srgb(&self) -> Self {
        // Rec.2020 -> XYZ -> sRGB
        let xyz = self.rec2020_to_xyz();
        let srgb = xyz_to_srgb(&xyz);
        Color { space: ColorSpace::Srgb, ..srgb }
    }
    
    fn p3_to_xyz(&self) -> [f64; 3] {
        // Linearize and apply P3 to XYZ matrix
        let lin_r = self.linearize(self.r as f64 / 1023.0);
        let lin_g = self.linearize(self.g as f64 / 1023.0);
        let lin_b = self.linearize(self.b as f64 / 1023.0);
        
        // P3 to XYZ matrix (D65 white point)
        [
            lin_r * 0.4865709 + lin_g * 0.2656677 + lin_b * 0.1982175,
            lin_r * 0.2289746 + lin_g * 0.6917385 + lin_b * 0.0792869,
            lin_r * 0.0000000 + lin_g * 0.0451134 + lin_b * 1.0439444,
        ]
    }
    
    fn linearize(&self, value: f64) -> f64 {
        // sRGB transfer function (approximate for P3)
        if value <= 0.04045 {
            value / 12.92
        } else {
            ((value + 0.055) / 1.055).powf(2.4)
        }
    }
}
```

## GPU Rendering

### Shader Changes

```wgsl
// Standard 8-bit color input
struct ColorInput8 {
    r: u8,
    g: u8,
    b: u8,
    a: u8,
}

// Deep color input
struct ColorInput16 {
    r: u16,
    g: u16,
    b: u16,
    a: u16,
    space: u32,  // 0=sRGB, 1=Display-P3, 2=Rec.2020
}

// For HDR displays
@fragment
fn fragment_deep(input: VertexOutput) -> @location(0) vec4<f32> {
    // Normalize from u16
    let color = vec4<f32>(
        f32(input.color.r) / 65535.0,
        f32(input.color.g) / 65535.0,
        f32(input.color.b) / 65535.0,
        f32(input.color.a) / 65535.0,
    );
    
    // Convert to output color space
    let output = convert_color_space(color, input.color.space);
    
    return output;
}

fn convert_color_space(color: vec4<f32>, from_space: u32) -> vec4<f32> {
    // Pipeline: from_space -> XYZ -> sRGB (display assumed)
    // For actual HDR output, need display to support extended color
    match from_space {
        0u => color,  // Already sRGB
        1u => p3_to_srgb(color),
        2u => rec2020_to_srgb(color),
        _ => color,
    }
}
```

### Theme Loading

```rust
fn load_theme(path: &Path) -> Result<Theme> {
    let content = fs::read_to_string(path)?;
    let raw: toml::Value = toml::from_str(&content)?;
    
    let mut theme = Theme::default();
    
    // Parse colors with optional bit depth
    for (name, color_value) in raw.get("colors").unwrap().as_table().unwrap() {
        let color = parse_extended_color(name, color_value)?;
        theme.colors.insert(name.clone(), color);
    }
    
    Ok(theme)
}

fn parse_extended_color(name: &str, value: &toml::Value) -> Result<Color> {
    let table = value.as_table().ok_or("color must be table")?;
    
    let r = parse_channel(table.get("r"))?;
    let g = parse_channel(table.get("g"))?;
    let b = parse_channel(table.get("b"))?;
    let a = table.get("a").and_then(|v| parse_channel(Some(v)).ok()).unwrap_or(1023);
    
    let space = table.get("space")
        .and_then(|v| v.as_str())
        .map(|s| match s {
            "srgb" => ColorSpace::Srgb,
            "display-p3" => ColorSpace::DisplayP3,
            "rec2020" => ColorSpace::Rec2020,
            _ => ColorSpace::Srgb,
        })
        .unwrap_or(ColorSpace::Srgb);
    
    Ok(Color { r, g, b, a, space })
}

fn parse_channel(value: Option<&toml::Value>) -> Result<u16> {
    match value {
        Some(toml::Value::Integer(i)) => {
            // Accept 0-255 (8-bit) or 0-1023 (10-bit) or 0-4095 (12-bit)
            let v = *i as u16;
            Ok(if v <= 255 { v << 8 } else { v })
        }
        Some(toml::Value::String(s)) => {
            // Parse hex: "#FF" or "#FFF" or "#FFFF"
            parse_hex_channel(s)
        }
        _ => Err("invalid color channel".into()),
    }
}
```

## Notes

### Compatibility

- Standard terminals only support 24-bit color
- Deep color is a forward-looking feature
- Fallback to 8-bit for non-HDR displays
- Most themes use 8-bit colors anyway

### Priority

This is P3 (low priority) because:
1. HDR displays are not yet common
2. Limited escape sequence support
3. Most users won't notice difference
4. Can be added after core functionality

### Future

When HDR becomes standard:
- Add OLED burn-in protection
- Support Dolby Vision / HDR10 metadata
- Auto-detect display capabilities
- Per-monitor color profiles
